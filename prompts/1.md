매물 점검 페이지 만든다.
주 용도는 해당 매물 정보를 입력했을 때 점검 결과를 토해내는 것이다.

현재로서 점검할 수 있는 건
1. 주택 시세 확인 - 보증금이 주택 시세 80% 넘는지
2. 건축물대장 정보 - 이는 이미 api로 가져오는 것 구현 되어 있다.  위반 건축물 여부, 건물 용도가 주거용 맞는지
이 두 개를 점검한 결과를 보여주는 것이다.

현재는 그냥 정보 보여주는 것에 그치는데, 이 정보들을 조합해 LLM이 분석 결과를 자연어로 설명해주면 좋겠다. 이는
system prompt에 규칙을 위 두 규칙을 점검하라고 하면 된다.
그리고 현재는 '주택 시세, 건축물대장 정보' 두 개지만 나중에 (예: 계약서?) 새로운 정보들이 추가되더라도 확장 가
능한 구조로 만들어라.


## 1. 주택 시세 비교
필요한 건 두 개다. 아파트로 예를 들었을 때,
1) 아파트 실거래가
2) 아파트 전세 보증금

현재 2) 아파트 전세 보증금은, 지도에서 도, 시와 구, 동을 입력하면 각 매물의 전/월세 매매 거래가가 나온다. 여기에서 사용하는 api에서 오는 결과 XML에서 목표로 하는 아파트 이름 등을 타겟해서 검색하면 될 것이다.
지금 1) 아파트 실거래가를 가져오기 위해 2)와 동일하게 공공데이터포털에 api 사용 신청을 했다. 그런데 아직 신청 허가가 안 났는지 사용이 힘든데,
그래서 내가 원하는 것은 1) 아파트 실거래를 가져올 수 있다고 치고 (거의 동일한 메커니즘이다. url에 api key와 함께 요청 보내서, 매물 이름으로 원하는 내용 필터링), 시세와 전세 보증금 정보를 LLM에게 건네주는 것이다.


## 2. 건축물대장 정보
이것도 여러 후보가 검색되는데, 여기서 원하는 매물(예: 아파트 이름)으로 필터링해서 기본 정보 LLM에게 건네주도록 하자.
