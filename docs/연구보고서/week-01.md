# Week 01

## 목표

- 프로젝트의 문제 정의와 1차 기능 범위를 정리한다.
- 지도 기반 건축물대장 조회 흐름을 연결한다.
- 전세사기 예방 관점의 건축물대장 체크 포인트를 정리한다.

## 사용 데이터 / API

- Naver Geocoding API
- Naver Reverse Geocoding API
- 국토교통부 건축물대장정보 서비스 `getBrTitleInfo`

## 진행 내용

- 지도 페이지에서 주소 검색과 지도 클릭을 통해 건물을 선택하는 흐름을 구현했다.
- 주소 검색은 바로 확정하지 않고, 후보 목록을 먼저 보여준 뒤 사용자가 직접 선택하도록 변경했다.
- 선택된 건물의 건축물대장 핵심 정보를 우측 패널에 표시하도록 구성했다.
- 건축물대장 정보 중 전세사기 예방에 유의미한 항목을 정리한 접이식 가이드를 추가했다.
- 건축물대장 조회를 위한 임시 스크립트를 추가하고, 공공데이터 API 응답을 확인했다.

## 이슈와 해결

- 주소를 좌표로 바꾼 뒤 자동으로 건물을 확정하면, 실제 주소와 다른 후보가 선택되는 문제가 있었다.
- 이를 해결하기 위해 자동 확정 방식을 버리고, 후보 목록을 스크롤로 노출한 뒤 수동 선택 방식으로 바꿨다.
- 법정동 코드와 지번 정보가 맞지 않아 조회가 비어 보이는 경우가 있었고, reverse geocoding 결과를 기반으로 건축물대장 조회 조건을 조정했다.
- 건축물대장 응답은 원문 그대로보다 화면용 요약 정보가 먼저 필요해, 프론트에 필요한 필드만 추려서 보여주도록 정리했다.

## 결과

- 지도에서 주소 검색, 후보 선택, 건축물대장 요약 확인까지 이어지는 기본 흐름을 만들었다.
- 전세사기 예방 관점에서 건축물대장 확인 포인트를 사용자에게 안내할 수 있는 UI를 추가했다.

## 남은 과제

- 후보 매칭 정확도를 더 높일 필요가 있다.
- 건축물대장 상세 필드와 실제 계약 판단에 필요한 정보의 우선순위를 더 정교하게 정리해야 한다.
- 지도 클릭 방식과 주소 검색 방식의 UX를 더 다듬을 필요가 있다.
