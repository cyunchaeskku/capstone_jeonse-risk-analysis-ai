# Document Pipeline Spec

## 목적

여러 입력 소스에서 데이터를 추출, 정규화, 저장하는 파이프라인 기준을 정의한다.

## 범위

- 문서 업로드 수신
- 텍스트 추출 및 구조화
- 공공 데이터 수집 및 정제
- 분석 엔진과 QA 계층에 필요한 저장 형식 제공

## 입력 소스

- 계약서 PDF
- 등기부등본 PDF 또는 이미지
- 공공 API 데이터
- 수기 입력 텍스트
- `legalize-kr` 저장소 기반 법령 원문 데이터

## 처리 단계 초안

1. 원본 수신 및 메타데이터 기록
2. 문서 유형 판별
3. 텍스트 추출 또는 OCR 수행
4. 필드 정규화 및 구조화
5. 정형 저장소와 검색 저장소에 분리 저장

## 외부 소스 메모

- `legalize-kr`
  - URL: `https://github.com/legalize-kr/legalize-kr/tree/main`
  - 용도: 법령 원문 수집 및 전처리 후보 소스
  - 비고: 실제 ingest 단위, 업데이트 주기, 저장 형식은 별도 결정 필요

## 산출물

- 정규화된 텍스트
- 구조화 필드
- 파싱 신뢰도 또는 오류 정보
- 후속 분석용 참조 ID

## 실패 및 예외 시나리오

- 스캔 품질 저하
- 문서 유형 오분류
- 외부 API 실패
- 문서 일부만 추출된 경우

## 결정 필요 항목

- OCR fallback 도입 여부
- 원본 파일 저장 위치와 보관 기간
- 재처리 전략과 idempotency 기준
