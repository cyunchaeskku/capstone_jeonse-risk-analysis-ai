# Legal QA Spec

## 목적

법률 및 판례 기반 질문 응답 기능의 동작 기준을 정의한다.

## 범위

- 질문 해석
- 질문 분류 라우팅
- 관련 문서 검색
- 근거 기반 답변 생성
- 출처 제시

## 비범위

- 위험도 자체를 재판정하는 기능
- 법률 자문 대체

## 입력 초안

- 사용자 질문
- 선택적 분석 컨텍스트
- 선택적 문서 컨텍스트

## 출력 초안

- 답변 텍스트
- 근거 문서 목록
- 관련 조문 또는 판례 식별자
- 참고 및 면책 문구

## 초기 구현 범위

- v1은 LangGraph + LangChain + OpenAI 기반 챗봇으로 구현한다.
- 질문을 먼저 분류해 단순 질문은 바로 답변하고, 법령 질문만 벡터 검색으로 분기한다.
- 법령 검색은 `vectorDB/laws_faiss`의 로컬 FAISS 인덱스를 사용한다.
- 프론트는 최근 2세트의 대화 이력만 `history`로 전달한다.
- 분석 결과 ID가 있으면 참고 컨텍스트로만 사용하고 위험 판정을 다시 하지 않는다.
- 챗봇은 전세 계약, 위험 확인 포인트, 일반적 법률/절차 설명에 집중한다.
- 범위 밖 질문은 짧게 제한하고 전세 관련 질문으로 유도한다.

## 검색 대상 초안

- 주택임대차보호법
- 공인중개사법
- 전세사기 관련 판례
- 공공기관 체크리스트 및 가이드

## 외부 소스 및 연동 후보

  - URL: `https://github.com/legalize-kr/legalize-kr/tree/main`

## 실패 및 예외 시나리오

- 질문 분류가 애매한 경우
- 관련 문서 검색 실패
- 상충하는 해석 존재
- 질문이 범위를 벗어나는 경우
- 최신성 확인이 필요한 법령 변경 이슈

## 결정 필요 항목

- 검색 단위를 문단 기준으로 할지 조문 기준으로 할지 여부
- 분석 결과와 QA 컨텍스트를 자동 연결할지 여부
- 답변 톤과 면책 문구 수준
- MCP 실시간 조회와 사전 인덱싱 중 어떤 경로를 우선 경로로 둘지 여부
