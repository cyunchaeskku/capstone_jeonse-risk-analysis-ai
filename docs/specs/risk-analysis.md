# Risk Analysis Spec

## 목적

전세사기 위험 분석 기능의 입력, 처리, 출력 기준을 정의한다.

## 범위

- 규칙 기반 위험 요소 평가
- 개별 위험 요소 결과 생성
- 종합 위험도 산정
- 사용자 설명 생성에 필요한 구조화 결과 제공

## 비범위

- 법률 질의응답
- 장기 학습형 모델 기반 자동 분류

## 입력 초안

- 사용자 입력 매물 정보
- 계약 관련 수기 입력값
- 계약서 및 등기 문서 추출 결과
- 공공 데이터 기반 보조 정보

## 출력 초안

- 종합 위험도
- 개별 위험 요소 목록
- 각 위험 요소의 근거 데이터
- 사용자용 설명 생성 입력

## 핵심 규칙 카테고리 초안

- 전세가율
- 근저당 비율
- 권리관계
- 계약 특약 및 조건
- 보증보험 가입 가능성

## 초기 구현 범위

- FastAPI 연결 확인용 최소 규칙만 포함한다.
- 현재 예시 규칙:
  - 보증금 규모에 따라 `low`, `medium`, `high` 중 하나의 위험 요소를 생성한다.
  - 업로드 문서가 없으면 문서 근거 부족 위험 요소를 추가한다.
- 종합 위험도는 개별 요소 중 가장 높은 등급으로 집계한다.
- 이 구현은 데모용 부트스트랩이며 실제 전세 위험 분석 규칙의 완성본이 아니다.

## 매물 점검 v1 규칙

- 점검 규칙은 모듈 단위로 추가 가능한 구조를 유지한다.
- 현재 규칙:
  - `deposit_to_market_ratio`: 보증금/시세가 0.8 초과인지 점검
  - `residential_use`: 건축물대장 용도가 주거용인지 점검
- 제외 규칙:
  - `위반건축물 여부`는 데이터 소스 연결 전까지 v1에서 노출/판정하지 않는다.
- 설명 계층:
  - LLM은 규칙 결과 설명만 수행하며, 판정값 자체는 변경하지 않는다.

## 실패 및 예외 시나리오

- 문서 파싱 실패
- 필수 입력값 누락
- 상충하는 데이터 소스 존재
- 계산 불가능한 경우

## 결정 필요 항목

- 종합 위험도를 점수형으로 표현할지 등급형으로 표현할지 여부
- 규칙 실행 순서와 우선순위
- 필수 규칙 세트의 최소 범위
