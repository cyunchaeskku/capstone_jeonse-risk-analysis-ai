# Domain Model

## 목적

이 문서는 핵심 도메인 엔티티와 각 데이터의 의미를 정의한다.

## 핵심 엔티티

### Listing

- 전세 분석 대상 매물
- 포함 후보: 주소, 건물 유형, 보증금, 시세, 전세가율 계산에 필요한 값

### ContractDocument

- 사용자가 업로드한 계약서 원문 및 추출 결과
- 원문 파일 메타데이터와 파싱된 텍스트를 함께 관리한다

### RegistryDocument

- 등기부등본 원문 및 추출 결과
- 소유권, 근저당, 권리관계 관련 위험 판단 입력값을 포함한다

### RiskRule

- 위험 요소를 판정하는 조건 정의
- 입력 조건, 판정 기준, 심각도, 설명 템플릿을 가진다

### RiskEvaluation

- 개별 규칙 평가 결과
- 포함 후보: 규칙 ID, 결과 상태, 근거 데이터, 메시지, 심각도

### RiskReport

- 하나의 분석 요청에 대한 종합 결과
- 개별 규칙 결과, 종합 위험도, 생성 시각, 설명 텍스트를 포함한다

### LegalDocument

- 법률, 시행령, 판례, 공공 체크리스트 등 QA 검색 대상 문서
- 출처, 발행일, 조문 또는 사건번호 같은 식별자를 포함한다

### QaSession

- 사용자 질문과 응답 이력
- 관련 검색 문서와 답변 근거를 연결한다

## 엔티티 관계 초안

- 하나의 `RiskReport`는 여러 `RiskEvaluation`을 가진다.
- 하나의 `RiskReport`는 하나 이상의 입력 문서와 연결될 수 있다.
- 하나의 `QaSession`은 여러 `LegalDocument`를 근거로 참조할 수 있다.

## 정리 필요 항목

- 매물 단위와 분석 요청 단위를 분리할지 여부
- 업로드 원문 보관 범위와 만료 정책
- 사용자 계정 도입 시 소유권 모델
